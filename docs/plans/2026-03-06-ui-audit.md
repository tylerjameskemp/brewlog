# UI Audit — BrewLog Cleanup & Simplification

**Created:** 2026-03-06
**Status:** In Progress
**Goal:** Get the app from "functional prototype" to "beta-ready" by pruning clutter, fixing friction, and polishing what's left.

## Guiding Principles

- **Subtractive first.** Remove before redesigning. Every element must earn its place.
- **One concept, one control.** No duplicate pickers, no competing UI for the same thing.
- **Arm's-length readability.** If you can't glance at it mid-pour, the text is too small.
- **Always tappable.** No "edit mode" toggles. If it's on screen, you can interact with it.
- **Mobile-first for real.** iPhone 13 Mini is the baseline. If it works there, it works everywhere.

## User Context

- Buys new beans frequently (not dial-in focused)
- Brews 1-2 cups/day, wants fast "confirm and go" flow
- Rate/tasting screen is out of scope for now
- Approaching beta-tester readiness

## Phase 1: Prune (Subtractive Pass)

_Goal: Make the app smaller. Cut everything that doesn't earn its place. No new features._

### 1.1 RecipeAssembly Screen (biggest pain point)

| # | Item | What to do | Why |
|---|------|-----------|-----|
| 1.1a | Origin Details swipe card | **Remove entirely** | Origin/process/roaster is bean library data, not brew-time data. Nobody checks origin mid-brew. |
| 1.1b | Swipe card mechanism | **Remove** — show essentials inline (no cards, no dots, no swiping) | Cards hide information behind a swipe gesture that users don't discover. Just show the fields. |
| 1.1c | Edit/Done toggle | **Remove** — fields always tappable | Extra tap for no reason. You said it: "you shouldn't have to press edit first." |
| 1.1d | Separate template picker page | **Remove** — fold templates into recipe dropdown | "Choose a pour template" navigates to a whole new page. Recipes and templates should be one concept. |
| 1.1e | "Choose a pour template" link below steps | **Remove** | Redundant with template/recipe picker above. |
| 1.1f | "Revert steps to standard 3-pour V60" link | **Remove** | Niche action cluttering the screen. User can pick a different recipe instead. |
| 1.1g | Pour Templates horizontal bar | **Remove** (consolidated into recipe picker) | Third place to pick a template. One picker, not three. |
| 1.1h | Equipment section | **Collapse by default** | Rarely changed brew-to-brew. Still accessible, just not in your face. |
| 1.1i | "Save changes to recipe" button | **Move or rethink** | Currently floating mid-screen. Should be a prompt after action, not a persistent button. |
| 1.1j | Pour Steps section | **Collapse by default**, show step count summary | Most brews reuse the same steps. Show "4 steps · 240g" collapsed. |

**Expected result:** RecipeAssembly goes from ~14 sections to ~5-6: bean card, recipe picker, brew params (inline), collapsed steps, collapsed equipment, Brew This button.

## Phase 2: Fix Friction

_Goal: Make what's left work smoothly. Fix the interactions that fight the user._

| # | Item | Notes |
|---|------|-------|
| 2.1 | Teleprompter text size | Bump current step name + water target significantly. Readable at arm's length. |
| 2.2 | Teleprompter "up next" visibility | After tapping a step, make the next step more prominent (not just the same faded list). |
| 2.3 | Target time range on brew screen | Show green/amber/red indicator for "in range" vs "over/under" during brew. |
| 2.4 | Bean picker — recipe preview | When selecting a bean, show a 1-line recipe summary (e.g., "Last: 16g/272g, grind 5, 3:30") so you know what recipe you'll get. |
| 2.5 | History — stop truncating names | Let bean names wrap or use a smarter layout. |
| 2.6 | History — simplify expanded view | Show essential brew info first, tasting details behind a "more" tap. |
| 2.7 | Pour step water scaling | When dose/water changes, offer to proportionally scale step water targets. |
| 2.8 | Recipe + template consolidation (logic) | Templates become "starter recipes" — when you pick one, it creates a recipe for that bean. One concept everywhere. |
| 2.9 | Bean rename → recipe linking | Renaming a bean should update recipe references too (currently breaks). |
| 2.10 | "Notes from Last Brew" placement | Currently below recipe dropdown but above pour templates — needs to be near the top, it's the most useful thing on screen. |
